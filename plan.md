# CORTEX.md Installation Architecture Plan

## Problem Statement

When cortex is installed as a **Claude plugin**, the plugin's internal `CLAUDE.md` is never read because Claude only auto-discovers:
1. `~/.claude/CLAUDE.md` (global user config)
2. `<project-root>/CLAUDE.md` (project-specific config)

Plugins are sandboxed—their internal `.md` files are invisible to Claude's context loader unless explicitly referenced via `@` directives from one of the two discovered locations.

## Claude's Directory Loading Behavior

### Auto-Discovered (both ~/.claude AND plugins)
These directories are automatically loaded by Claude without needing `@` references:
- `agents/` - Agent definitions
- `commands/` - Slash commands
- `skills/` - Skill definitions
- `hooks/` - Hook scripts
- `.mcp.json` - MCP server configuration
- `.lsp.json` - LSP server configuration

### Require Explicit @ References (never auto-discovered)
These must be referenced via `@` directives in CLAUDE.md or a file it references:
- `modes/` - Behavioral modes
- `flags/` - Flag configuration files
- `mcp/docs/` - MCP server documentation

### Special Case: `rules/`
**Critical Asymmetry:**
- **~/.claude mode**: `rules/*.md` files are loaded recursively and automatically
- **Plugin mode**: `rules/` is NOT auto-discovered; must use explicit `@` references

This asymmetry means we need a **unified rules architecture** that works elegantly in both modes while giving users fine-grained control.

## Proposed Solution

Rename the framework entry point from `CLAUDE.md` → `CORTEX.md` and establish two installation modes:

### Mode 1: Claude Plugin Installation
```
~/.claude/
├── CLAUDE.md          # User's existing/new file
│   └── @CORTEX.md     # Single line reference added
└── CORTEX.md          # Symlink → <plugin-path>/CORTEX.md
```

### Mode 2: Standalone Installation (non-plugin)
```
<project-root>/
├── CLAUDE.md          # Simple redirect file
│   └── @CORTEX.md
├── CORTEX.md          # Full framework entry point
├── FLAGS.md           # Flags configuration
├── PRINCIPLES.md      # Auto-generated from principles/
└── RULES.md           # Base behavioral rules
```

## Architecture Decision: Symlink vs Copy

**Recommendation: Symlink for plugin mode**

| Aspect | Symlink | Copy |
|--------|---------|------|
| Updates | Automatic when plugin updates | Requires manual re-copy |
| Disk space | Minimal | Duplicates content |
| Portability | May break if plugin moves | Always works |
| User customization | Can't customize without breaking link | Full control |
| Maintenance | Zero | Must track versions |

**Decision**: Use symlink with fallback to copy for Windows or if symlink fails.

## Installer Changes

### New Installation Flow

```
cortex install --plugin
    │
    ├─→ Detect installation mode
    │   ├─→ Plugin mode? (--plugin flag or detect plugin location)
    │   │   ├─→ Create symlink: ~/.claude/CORTEX.md → <plugin-path>/CORTEX.md
    │   │   ├─→ Append "@CORTEX.md" to ~/.claude/CLAUDE.md (create if needed)
    │   │   └─→ Copy supporting files (rules/, modes/, agents/, etc.) to ~/.claude/
    │   │
    │   └─→ Standalone mode? (default)
    │       ├─→ Copy CORTEX.md, FLAGS.md, PRINCIPLES.md, RULES.md to project root
    │       ├─→ Create CLAUDE.md redirect in project root
    │       └─→ Copy supporting directories to project .claude/
    │
    └─→ Show post-install instructions
```

### New CLI Commands

```bash
# Plugin installation (symlink CORTEX.md to ~/.claude/)
cortex install framework --plugin

# Standalone installation (copy to current project)
cortex install framework --standalone

# Interactive mode (asks user which mode)
cortex install framework

# Check installation status
cortex install status
```

### Installer Module Updates

**File**: `claude_ctx_py/installer.py`

Add new function:

```python
def install_framework(
    mode: Literal["plugin", "standalone", "auto"] = "auto",
    target_dir: Optional[Path] = None,
    force: bool = False,
    dry_run: bool = False,
) -> Tuple[int, str]:
    """Install CORTEX.md framework files."""

    if mode == "auto":
        mode = _detect_installation_mode()

    if mode == "plugin":
        return _install_plugin_mode(force, dry_run)
    else:
        return _install_standalone_mode(target_dir or Path.cwd(), force, dry_run)


def _install_plugin_mode(force: bool, dry_run: bool) -> Tuple[int, str]:
    """Install framework as Claude plugin (symlink to ~/.claude/)."""
    plugin_root = _find_plugin_root()  # Find where cortex is installed
    claude_home = Path.home() / ".claude"

    # 1. Create symlink for CORTEX.md
    cortex_source = plugin_root / "CORTEX.md"
    cortex_target = claude_home / "CORTEX.md"

    if not dry_run:
        cortex_target.symlink_to(cortex_source)

    # 2. Add @CORTEX.md to CLAUDE.md
    claude_md = claude_home / "CLAUDE.md"
    if not dry_run:
        _ensure_cortex_reference(claude_md)

    # 3. Copy supporting directories (rules/, modes/, agents/, etc.)
    # ... (symlink or copy based on user preference)

    return 0, "✓ Installed cortex framework in plugin mode"


def _ensure_cortex_reference(claude_md: Path) -> None:
    """Ensure CLAUDE.md contains @CORTEX.md reference."""
    if claude_md.exists():
        content = claude_md.read_text()
        if "@CORTEX.md" not in content:
            # Add reference after any existing header
            lines = content.split("\n")
            insert_pos = _find_insert_position(lines)
            lines.insert(insert_pos, "@CORTEX.md")
            claude_md.write_text("\n".join(lines))
    else:
        # Create new CLAUDE.md with just the reference
        claude_md.parent.mkdir(parents=True, exist_ok=True)
        claude_md.write_text("# Claude Framework Entry Point\n\n@CORTEX.md\n")
```

## Edge Cases & Error Handling

### Symlink Failures
- **Windows without developer mode**: Fall back to copy with warning
- **Permission denied**: Suggest running with elevated permissions
- **Target exists**: Respect `--force` flag or prompt user

### Existing CLAUDE.md
- **Already has @CORTEX.md**: Skip modification, report no-op
- **Complex user config**: Append reference at end, don't overwrite
- **Read-only file**: Error with clear message

### Plugin Location Detection
- Check `CLAUDE_PLUGIN_ROOT` env var first
- Then check common plugin paths
- Fall back to package location via `sysconfig`

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Symlink breaks on plugin update | Low | Medium | Use relative path, document expected location |
| Windows symlink issues | Medium | Medium | Fallback to copy, clear error message |
| Overwrite user CLAUDE.md | Low | High | Always append, never replace existing content |
| Claude doesn't resolve symlinks | Low | High | Test early, have copy fallback ready |
